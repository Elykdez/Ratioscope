# Ratioscope 使用指南

## 简介

Ratioscope 是一件艺术作品：

- 它以实时展开的机器语言为主题进行视觉研究。
- 它把已经完成的模型前向计算和下一个 token 的概率转化成不断变化的发光场。
- 画面中的部分数值来自真实的模型遥测数据，但它们的空间构图属于艺术表达，并不是生物神经元地图。

同时也是一个可实际使用的本地纯文本聊天机器人：

- 以及语言模型推理过程的实时可视化。
- 通过 Unity Inference Engine 运行量化后的 Qwen3 模型。分词、对话历史、文本生成和 Cortex 画面都在本机完成- 运行时不需要第三方服务或云端模型。

## 启动模型

Ratioscope 会根据可用显存选择模型配置，并在应用打开后立即开始加载。

- 在高显存系统上，它选择具有 4096-token 窗口的 4B 模型和 GPUCompute 后端。
- 在中等显存的 GPU 上，它选择具有 2048-token 窗口的 1.7B 模型和 GPUCompute 后端。
- 当显存有限或没有可用 GPU 时，它选择在 CPU 上运行的 1.7B 模型。也可以通过手动配置使用 4B CPU 模式。

加载遮罩会显示所选模型文件、后端、文件大小和已经等待的时间。加载完成前，**SEND** 按钮不可用。当状态以 **ready** 开头时，还会显示当前配置、模型文件、后端、Transformer block 数量、上下文窗口和加载时间。顶部的 **FPS** 和 **VRAM** 分别显示当前渲染帧率与显存使用情况。

生成后的模型文件体积较大，因此启动可能需要一些时间。如果状态显示 **model load failed**，通常表示所选 `.sentis` 文件缺失或无法读取、设置配置中的文件名不匹配，或者模型无法装入可用内存。

## 开始聊天

1. 等待状态变为 **ready**。
2. 在 Cortex 下方的输入框中输入消息。
3. 按 **Enter** 或点击 **SEND** 发送。使用 **Shift+Enter** 可以换行而不发送。
4. 回复会逐 token 出现在对话区，同时 Cortex 会把本次计算转化成动态画面。

同一时间只能执行一次回复生成或上下文压缩。模型工作期间，发送和压缩按钮会暂时禁用。状态栏会显示计算图进度、已生成 token 数、每个 token 的耗时、上下文占用和熵。生成完成后，状态栏会显示回复长度和总耗时。

当前界面每次回复最多请求生成 512 个新 token。如果模型提前输出结束标记，或者上下文窗口剩余空间不足，实际回复会更短。1.7B 配置支持单独的 thinking 阶段：推理过程会流式显示在弱化的 **thinking** 气泡中，可见回答则显示在 **ai** 气泡中。不支持 thinking 的配置只显示回答。

模型可能给出错误事实。当前版本不支持联网搜索、工具调用、图片输入、音频输入或外部知识查询。

## 阅读 Cortex

Cortex 包含两个数据区域。

- **Transformer 结构。** 1.7B 模型以 56 行显示全部 28 个 block；4B 模型以 72 行显示全部 36 个 block。每个 block 包含一行 attention 和一行 MLP，并被划分为带标签的架构阶段。每次真实的模型 forward 完成后，完整结构都会脉冲发亮。
- **候选 token。** 每生成一个 token，模型领先的下一 token 候选会在 token 区域闪亮。单元格越亮，候选概率越高。
- **颜色：不确定性。** 熵较低时画面更接近绿色；熵较高时会转向蓝色。高熵表示模型当时在更多可能的下一 token 之间进行选择。
- **衰减：近期活动。** 热度会随时间逐渐消退，使刚完成的 forward 和刚出现的候选在画面上短暂保留。

把鼠标移到任意 Cortex 单元格上可以查看详情。结构单元格会显示 Transformer block、attention/MLP 分支、架构阶段和近期参与热度；token 单元格可以显示解码后的 token 及其概率。提示框还会显示当前熵以及最多五个领先候选 token。

## 哪些是真实数据，哪些是艺术表达

显示的 block 数量来自已加载模型的 K/V 缓存接口；对于没有缓存接口的计算图，则使用明确配置的回退值。Forward 完成事件、候选 token、候选概率、熵、耗时和上下文占用都来自正在运行的模型。缓存计算图为每个 Transformer block 暴露一对 K/V，因此 Ratioscope 无需读取隐藏张量也能区分全部 28 或 36 个 block。

Attention/MLP 行内标注的阶段属于结构图示。结构区域的亮度记录稠密 Transformer 堆栈近期是否参与计算；它不代表隐藏状态或神经元激活强度，也不表示每个 block 的独立计时。Token ID 经过哈希后映射到显示位置，因此相邻单元格不表示语义相关。这些 Qwen 模型都是稠密 Transformer，没有 mixture-of-experts 路由网格。

因此，Cortex 是一幅以真实输出遥测为材料的结构性艺术表达，而不是观察隐藏状态或思想的显微镜。若要显示中间激活值，需要另外导出经过专门插桩的模型。

## 本地模型如何工作

Ratioscope 为两个 uint8 模型文件提供 CPU 和 GPU 配置：Qwen3-1.7B 使用 2048-token 窗口，Qwen3-4B-Instruct-2507 使用 4096-token 窗口。一个设置资源统一管理模型文件名、后端选择、显存分级策略、Transformer block 回退数量和 Cortex 外观。

文本每次生成一个 token。固定形状的 K/V 缓存计算图会在生成 token 和多轮对话之间保留之前的 attention key/value，使模型可以计算新 token，而不必重新计算整个窗口。1.7B 配置还保留了无缓存计算图作为兼容回退。上下文窗口大小仍由导出的模型固定；K/V 缓存可以提高复用效率，但不会扩大窗口容量。

## 模型配置

运行时模型清单位于 `StreamingAssets/config/models.json`。它配置可下载的模型文件：

- `baseUrl` 是模型下载的公共地址。除非某个 `relativeUrl` 已经是完整的 HTTP 或 HTTPS 地址，否则它会被追加到 `baseUrl` 后面。
- `entries` 通过 `fileName`、`byteSize`、`sha256` 和 `relativeUrl` 描述每个可下载的 `.sentis` 文件。

Cortex 的系统提示词在 **设置** 中编辑并与其他设置一起保存，面板中的 **恢复默认** 会还原该默认值，修改会在下一次对话开始时生效。

打开 **设置** 并选择 **下载模型**，即可下载清单中尚不可用的文件。下载内容会先以 `.part` 临时文件写入应用的持久化 `Models` 目录，并按照配置的文件大小和 SHA-256 校验值进行验证；只有验证通过后才会成为正式模型文件。已下载的文件优先于 `Assets/StreamingAssets/Sentis` 下同名的内置文件。`sha256` 留空会跳过哈希校验，但当 `byteSize` 大于零时仍会验证文件大小。

## 上下文与工作记忆

当前模型使用固定的 2048-token 或 4096-token 窗口。系统提示词、对话历史、当前消息、thinking 文本和回复共同占用这个窗口。

- 圆形百分比控件显示当前上下文占用率。
- 点击该百分比可以打开 **CONTEXT / WORKING MEMORY** 面板。
- 面板会显示提示词 token 数、回复可用空间、模型实际保留的消息、已生成的记忆，以及当前窗口之外的消息。
- 至少完成一轮“用户消息 + 助手回复”后，**COMPACT** 按钮才会可用。

上下文压缩必须手动触发。点击 **COMPACT** 后，当前正在使用的本地模型会把最早的一轮完整对话与已有压缩记忆总结成一条简短记忆，然后从活动历史中移除原始对话。这样可以让重要信息保留更久，但摘要由模型生成，仍可能遗漏或改变细节。

Ratioscope 不会自动压缩上下文。如果提示词本身已无法放入窗口，运行时可以移除最早的完整对话。如果系统提示词加上最新消息仍然过长，则无法发送。建议在百分比接近满载前使用 **COMPACT**，或者重启 Ratioscope 开始一个空白会话。

对话历史和压缩记忆只存在于本次运行中。关闭应用后，它们会被清除。

## 工具栏与更新

- **帮助** 按钮会打开本指南，并显示应用版本。
- **设置** 按钮会打开当前配置面板；当版本发布地址已经配置时，可以查看更新状态、手动检查更新并打开发布页面。

更新功能只检查是否存在新版本，并可以打开发布页面；它不会自动下载或安装更新。

## 隐私

聊天文本、分词、模型推理、上下文压缩和 Cortex 遥测数据都保留在本机。独立的更新检查器可能访问当前构建配置的版本发布地址，但不会发送任何对话内容。

## 常见问题

- **加载画面长时间显示：** 大型模型文件需要时间读取和准备。重启前先观察已经等待的时间和 VRAM 显示是否仍在变化。
- **出现 model load failed：** 确认所选配置指定的 `.sentis` 文件存在于 `Assets/StreamingAssets/Sentis`，并确认所选后端有足够内存。
- **回答很短：** 模型可能提前输出了结束标记，也可能是当前上下文窗口留给回复的空间太少。打开上下文面板并压缩较早的对话。
- **出现 cannot start 或提示词无法放入窗口：** 缩短最新消息、压缩一轮较早的完整对话，或者重启应用以清空会话。
- **COMPACT 按钮不可用：** 等待模型空闲，并先完成至少一轮用户与助手对话。
- **生成速度较慢：** 本地推理速度取决于所选模型、后端和硬件。状态栏中的每 token 耗时以及 FPS/VRAM 显示可以用来确认任务是否仍在推进。
